Use Service Management API of Azure Java SDK

Task 1 – Create a x509 Certificate for the Windows Azure Management API

In this task, you will learn how to Create a x509 Certificate for the Windows Azure Management API.

  1. Load the IIS 7 management console. I’m assuming here you have IIS7 installed since its required for the Windows Azure SDK.
  2. Click on your Server.
  3. Double Click Server Certificates in the IIS Section in the main panel.

  4. Click Create Self-Signed Certificate… in the Actions panel.

  5. Here We give it a Friendly Name democert.

  6. Close IIS Manager.
  7. Open Certificate Manager (Start->Run->certmgr.msc)
  8. Open Trusted Root Certification Authorities, then Certificates.
  9. Look for your certificate (Tip: Look in the Friendly Name column).

  10. Right Click your certificate, then choose All Tasks, then Export.
  11. In the Wizard, We have two choices about whether we export certificate with a private key or not. First, we choose No to export certificate without private key which is uploaded to Windows Azure Portal later.And second, we choose Yes to export certificate with private key which is used by our following process to generate a JKS format keystore.This keystore is used in Service Management API of Azure Java SDK.

    Choose No :

    • In the Wizard, choose No, do not export the private key, then click Next.

    • In this page, please choose the DER file format.Click Next button to step to next page.

    • Give your cert a name,then click Next.

    • Click Finish to start export process.

    • Now, we export a certificate file named democert.cer.

    Choose Yes :

    • In the Wizard, choose Yes, export the private key, then click Next.

    • In this page we keep the default value, then click Next.

    • Specify a password for protecting your private key,here we set our password as democert, then click the Next.

    • Specify a name for pfx file,then click Next.

    • Click Finish to start export process.

    • Now, we export another certificate file named as democert.pfx.

  12. Up to now, we have two certificate files which will be used later.

Task 2 – Upload certificate to your Azure account

  1. There are three ways you can upload certificates to your Azure account – the Azure Portal, Azure MMC Snap-in, and Azure Management Cmdlets. The steps below show how to upload using the Azure portal, which you can start by opening http://windows.azure.com in an IE window and login.  Then navigate to the Account tab and click the Manage My API Certificates link. 

  2. Browse to the democert.cer file and upload it. 

  3. Done.

    public static void convert(String pfxFile, String keystoreFile,
                String inputKey, String outputKey, String alias) throws Exception {
            File fileIn = new File(pkcs12file);
            File fileOut = new File(newjksfile);
            if (!fileIn.canRead()) {
                System.out.println("Unable to access input keystore: "
                        + fileIn.getPath());
                throw new Exception("Unable to access input keystore: "
                        + fileIn.getPath());
            }
            if (fileOut.exists() && !fileOut.canWrite()) {
                System.out.println("Output file is not writable: "
                        + fileOut.getPath());
                throw new Exception("Output file is not writable: "
                        + fileOut.getPath());
            }
            KeyStore kspkcs12 = KeyStore.getInstance("pkcs12");
            KeyStore ksjks = KeyStore.getInstance("jks");
            char[] inphrase = inputKey.toCharArray();
            char[] outphrase = outputKey.toCharArray();
            kspkcs12.load(new FileInputStream(fileIn), inphrase);
            ksjks.load((fileOut.exists()) ? new FileInputStream(fileOut) : null,
                    outphrase);
            Enumeration eAliases = kspkcs12.aliases();
            int n = 0;
            List <String> list = new ArrayList<String>();
            if (!eAliases.hasMoreElements()) {
                throw new Exception(
                        "Certificate is not valid. It does not contain any alias.");
            }
            while (eAliases.hasMoreElements()) {
                String strAlias = (String) eAliases.nextElement();
                System.out.println("Alias " + n++ + ": " + strAlias);
                if (kspkcs12.isKeyEntry(strAlias)) {
                    System.out.println("Adding key for alias " + strAlias);
                    Key key = kspkcs12.getKey(strAlias, inphrase);
                    Certificate[] chain = kspkcs12.getCertificateChain(strAlias);
     
                    if (alias != null)
                        strAlias = alias;
     
                    ksjks.setKeyEntry(strAlias, key, outphrase, chain);
                    list.add(strAlias);
                }
            }
            OutputStream out = new FileOutputStream(fileOut);
            ksjks.store(out, outphrase);
            out.close();
        }
    Now, we explain how to invoke this method in our demo.
    • Parameter pfxFile represents the absolute path of our democert.pfx file.
    • Parameter keystoreFile represents the absolute path of our target file democert.keystore .
    • Parameter inputKey represents the password of our private key of democert.pfx file.
    • Parameter outputKey represents the password of our democert.keystore.
    • Parameter alias represents our private key's alias.

    For instance,assume that our democert.pfx file is in the folder C:\Users\dev\Desktop\Test, so we can invoke this method as
